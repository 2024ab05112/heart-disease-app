Abstract

This report details the implementation of a scalable, production-ready Machine Learning solution for heart disease risk assessment. By integrating the Heart Disease UCI dataset with modern MLOps practices—including containerization, automated CI/CD pipelines, and cloud-native orchestration—this project demonstrates a robust lifecycle for predictive modeling in a healthcare context.

Project Overview

The primary objective of this project is to bridge the gap between experimental data science and production-grade software engineering. The solution leverages health data to predict patient risk, ensuring the model is reproducible, monitored, and easily deployable.

Traffic Management

The Nginx Ingress Controller handles external traffic, routing requests based on path prefixes. /api directs traffic to the FastAPI backend, while / serves the static React frontend assets.

Inference Pipeline

The backend integrates with MLflow to fetch the production-versioned model. It logs health metrics and inference stats to Prometheus for real-time monitoring via Grafana.

Data Science & Experimentation

Exploratory Data Analysis (EDA)

  • Visualizations: Histograms for feature distribution and correlation heatmaps to identify primary risk factors.
  • Key Insights: High correlation observed between target risk and features such as chest pain type (cp) and maximum heart rate (thalach).
  • Quality Check: Data was validated for missing values (none found) and categorical features were properly encoded.

MLOps: CI/CD & Deployment

Continuous Integration

Automated linting via flake8 and unit tests using pytest triggered on every Pull Request to the main branch.

Containerization

Optimized Docker builds pushed to Docker Hub, utilizing multi-stage builds to minimize image surface area.

Continuous Deployment

GitHub Actions triggers a rolling update on the AKS cluster, updating deployment manifests with the latest image tags.

Verification Workflow

  • Pipeline logs show green status for both Build and Deploy stages.
  • Single FQDN access for UI, API, and monitoring dashboards.
  • Real-time metric visualization confirming data scraping from Backend pods.
